# HTTP-сервер списка задач
Сервер реализует API для управления списком задач

## Запуск
Сам сервер использует стандартные библиотеки python, поэтому установка зависимостей для tuj запуска не требуется. Запустить можно например так (через bash):

python server.py

Сервер запустится на http://localhost:8000

## Тесты
Для старта автотестов используются "pytest" и "requests", таким образом требуется их предварительно установить (опять пример через bash):

pip install -r requirements.txt

Сами тесты (с подробным выводом) запускаются (в bash) так:

pytest -v

## Данные
Файл "tasks.txt" (с собственно самими задачами) создается автоматически при добавлении первой задачи. При наличии данных заранее можно создать файл с нижеуказанной структурой, однако, очевидно, это не очень желательно, ввиду возможности ошибок и, в целом, больших времязатрат

Пример структуры файла (json):
[
  {
    "id": 1,
    "title": "Сделать дз",
    "priority": "high",
    "isDone": false
  }
]
